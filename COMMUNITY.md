## 🛠️ Ways to Contribute

You don't have to write code to contribute. Here's how people are helping:

- **🐛 Bug hunters** — find and report issues
- **📝 Writers** — improve docs, FAQ, examples
- **🌍 Translators** — translate the UI and docs
- **🧪 Testers** — test on different repos, OS, and languages
- **🎨 Designers** — improve UI/UX, create demo assets
- **📣 Advocates** — share ArchAI, write blog posts, post demos

## 🏷️ Issue Labels Guide

| Label | Meaning |
|---|---|
| `good first issue` | Great for new contributors — well-scoped and documented |
| `help wanted` | Maintainers want community help on this |
| `bug` | Something is broken |
| `enhancement` | New feature or improvement |
| `documentation` | Docs-only change needed |
| `parser` | Related to the code parsing engine |
| `ai` | Related to the Gemini AI layer |
| `frontend` | Related to the Next.js UI |
| `performance` | Speed or memory optimization |

## 🎯 Good First Issues

Great places to start if you're new to ArchAI:

1. **Add Ruby language support** — extend `parser.py` with Ruby `.rb` parsing
2. **Add a loading skeleton** — show skeleton cards while the graph loads
3. **Improve error messages** — make API error responses more user-friendly
4. **Write more tests** — add test cases to `backend/tests/`
5. **Dark/light mode toggle** — add a theme switcher to the nav
6. **Export graph as PNG** — add a "Download Graph" button using React Flow's export API
7. **Add `robots.txt`** — basic SEO hygiene
8. **Improve the empty state** — design a better empty state for each workspace tab

Browse all `good first issue` tickets: [GitHub](https://github.com/dharmitshah/archai/issues?q=label%3A%22good+first+issue%22)

## 📜 Community Principles

1. **Be kind.** Everyone here is learning and building.
2. **Be specific.** Vague bug reports are hard to action. Include logs, versions, and steps.
3. **Be patient.** This is a passion project. Maintainers respond within 48 hours.
4. **Be constructive.** If something is broken, tell us how to fix it.
5. **Give credit.** If you build on someone's work, acknowledge it.
